变更属性

要创建一个新的变更，必须输入下列属性：

表 8.2.  变更属性

属性 必填字段 详情
Title标题 是 简短描述 / 变更名称
描述 否 变更的详细文字描述
理由 否 变更背后的理由的文本解释。回答这个问题：“如果不实施这个变更，最可能的后果是什么？”
类别 是 定义变更的类型，例如：“3-正常”。
影响 是 定义这个变更可能的效果或影响，例如“4-高”。
优先级 是 定义这个变更的优先级，例如“5-非常高”。
状态 是 当创建一个新的变更时，会自动设置状态。当修改一个已有变更时，变更创建人和变更经理可以手动设置状态。可用的状态和结果状态在集成的状态模式转换中定义，参阅“ 变更状态模式转换 ”一节。
客户请求日期 否 如果需要的话，这个属性可以通过“ChangeEdit编辑变更”、“ChangeAdd添加变更”和“ChangeZoom变更详情”模板的系统配置不激活。这个属性显示客户期望的实施日期。
附件 否 允许附加相关的文件和文档。

类别 - 影响 - 优先级矩阵

为确定一个变更的优先级，OTRS::ITSM支持变更创建人从集成的矩阵选择一个建议的优先级（基于选择的类别和输入的变更影响）。这个建议的优先级可由变更创建人随时覆盖。

图 8.4. 变更矩阵


The category, impact and priority values given upon installation can be customized in the General Catalog by the administrator. To do this, the menu item "General Catalog" should be selected in the OTRS::ITSM Administration interface.

  • 1-非常低

  • 2-低

  • 3-正常

  • 5-非常高

  • 2-低

  • 4-高

  • 5-非常高

ITSM::变更管理::优先级

安装完成后，OTRS::ITSM变更管理生成选择字段“优先级”的下列值：

  • 1-非常低

  • 2-低

  • 3-正常

  • 4-高

  • 5-非常高

在输入变更的基本数据后，实施变更的参与人员可在相关人员功能中定义。

图 8.7. 变更相关人员

表 8.3.  相关人员

属性 必填字段 详情
变更经理 是 给一个服务人员分配当前变更的变更经理权限。
变更创建人 是 定义创建和处理当前变更的服务人员。当创建一个新的变更时，当前服务人员自动成为变更创建人。
变更咨询委员会 否 定义一组包括服务人员和客户联系人的人员。

将变更链接到变更请求

通过OTRS框架著名的链接机制，系统允许一个变更链接到一个工单。这样变更起源如变更请求或问题就很容易识别出来。

图 8.8. 变更与RFC的关系


此外还可以从变更起源的故障或问题管理流程直接打开相关的变更。在这种情况下，系统会创建一个工单和变更的透明链接。可以在系统配置中限制只有授权的服务人员才能创建变更。还可以限制工单类型“RFC”只有变更管理授权的服务人员才能使用。请查看以下的系统配置选项：

  • Ticket -> Core::TicketACL - Ticket::Acl::Module###200-Ticket::Acl::Module

  • ITSM Change Management -> Core::ITSMChange - ITSMChange::AddChangeLinkTicketTypes

  • ITSM Change Management -> Core::ITSMChange - ITSMChange::RestrictTicketTypes::Groups

定义条件

OTRS::ITSM允许基于变更和/或工作指令的属性定义条件和操作。从系统管理前端可以为变更创建人激活/去激活这些属性。

可以在这里定义工作流程。例如，当取消一个工作指令时设置整个变更为“取消”或“请求审核/批准清除”的工作流程。

图 8.9. 变更-条件


需要注意的是，不是按按照某个顺序执行条件，而是按条件设置的顺序处理操作。

定义操作

建立条件后，OTRS::ITSM允许为当前变更或所有/一个工作指令定义任意数量的操作。

变更级别可用的操作

变更对象可以执行下面的操作。如果定义的条件为真，操作符“设置”使选择的属性设置为一个指定的值。另一方面，操作符“锁定”在条件为真时冻结选择的属性，例如不能进行手动变更。
