Управление Изменениями, согласно ITIL, это процесс прохождения этапов выполнения услуги, целью которого заключается в управлении ИТ изменениями, в том числе планирования, документирования и реализации после утверждения и оформления. Целью процесса является минимизация негативных последствий для ИТ-инфраструктуры, в частности, на критические сервисы, в результате поспешных или плохо управляемых изменений или поправок.

Требуемые компетенции

Внедрение OTRS::ITSM требует значительной технической компетенции и подготовки. До технической реализации, основные элементы процесса управления изменениями, такие, как потребные потоки работ/workflows, метрики, или отчеты, должны быть определены.

Технические требования

Следующее программное обеспечение, перечисленное ниже, необходимо для внедрения модуля Управления Изменениями:

  1. OTRS Framework, version 6.0.1 or higher

  2. ITSM "GeneralCatalog" package, version 6.0

  3. ITSM "ITSMCore" package, version 6.0

Следующие пакеты рекомендуются, но не обязательны, в зависимости от дальнейших целей по внедрению:

  • ITSM "ITSMIncidentProblemManagement" package, version 6.0

  • ITSM "ITSMServiceLevelManagement" package, version 6.0

  • ITSM "ITSMConfigurationManagement" package, version 6.0

  • ITSM "ImportExport" package, version 6.0
